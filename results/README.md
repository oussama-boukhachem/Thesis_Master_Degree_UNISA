# Experimental Results

This directory contains the definitive outputs of the benchmarking campaign, encompassing both the raw execution data generated by the Java Microbenchmark Harness (JMH) and the quantitative reports produced during the statistical analysis phase.

## Directory Structure

The raw execution data is divided into two primary conditions, representing the two phases of the experiment:

### 1. `no_regression/`
This directory contain the baseline metrics. These tests were executed on the pure Apache Ignite codebase without any injected performance anomalies.

### 2. `regression/`
This directory contains the post-injection metrics. Prior to these test runs, a synthetic performance regression (`Blackhole.consumeCPU(1000000L);`) was injected to simulate a software performance degradation.

Within both the `no_regression/` and `regression/` directories, the data is further segmented to allow for a one-to-one comparison:
* **By Commit:** Data is isolated into `commit1_b038730`, `commit2_feba953`, and `commit3_160dab0`.
* **By Origin:** Inside each commit folder, results are strictly split into `handwritten_results/` (the human-developer control group) and `ai_results/` (the Large Language Model experimental group).

## Raw Data Format

All raw JMH output is stored as `.json` files (for eg`JmhCacheBenchmark_get.json`). 

Instead of relying solely on the human-readable text summaries normally printed to the terminal, we forced JMH to dump its results in JSON format. This preserves the raw dataset of every single iteration across every fork. Preserving this granularity was strictly required to perform the Hierarchical Bootstrap statistical methodology, which resamples data at both the fork and iteration levels to calculate rigorous Confidence Intervals (CIs).

## Statistical Output Files

In addition to the raw JSON payloads, this directory contains the final outputs generated by the python analysis scripts:

* **`hierarchical_bootstrap_results.txt`**: The complete, detailed log of the bootstrap analysis. It contains the calculated confidence intervals, variance metrics, and p-values for every individual benchmark compared against its baseline.
* **`hierarchical_bootstrap_summary.txt`**: An aggregated, high-level overview summarizing the performance of the AI-generated benchmarks versus the handwritten benchmarks. It details metrics such as the total percentage of accurately detected regressions.
* **`confirmed_regressions.txt`**: A definitive log that cross-references the expected regressions with the statistically significant findings validated by the bootstrap method.

## Usage

The `.json` files stored within the `no_regression/` and `regression/` directories act as the exact input dataset for the `hierarchical_bootstrap.py` script located in the `scripts/` folder. If you wish to reproduce the statistical analysis, ensure these JSON files are properly placed before running the evaluation script.
