# Execution and Analysis Scripts

This directory contains the automation scripts developed to ensure a reliable and reproducible experimental environment. These scripts manage the system configuration, the injection of code regressions, the execution of the benchmarks, and the final statistical analysis.

## Overview of Scripts

### System Environment Configuration
To minimize external noise and system interference during benchmark runs, we use dedicated scripts to stabilize the operating system features.
* **`optimize_system.sh`**: Prepares the OS for benchmarking by disabling Turbo Boost, hyper-threading, and ASLR (Address Space Layout Randomization). This helps ensure CPU performance is flat and reproducible across test runs. 
* **`restore_system.sh`**: Reverses the changes made by the optimization script, returning the operating system back to its normal, everyday state once the benchmarking phase is completed.

### Regression Management
To simulate a performance degradation inside the benchmark methods, we use bash scripts that automate the source code editing process across the entire repository.
* **`inject_regression_code.sh`**: A script that parses the directory tree to find Java methods annotated with `@Benchmark`. It automatically injects `org.openjdk.jmh.infra.Blackhole.consumeCPU(1000000L);` exactly after the method opens, representing a performance regression for our testing phase.
* **`prepare_no_regression.sh`**: Comments out the previously injected `Blackhole.consumeCPU(1000000L);` method calls throughout the codebase to restore the clean baseline state for baseline execution.

### Benchmark Execution
These scripts orchestrate the execution of the Java Microbenchmark Harness (JMH) frameworks.
* **`run_finals_no_regression.sh`**: Executes the clean, baseline benchmarks across all three Apache Ignite commits. It configures the JVM (allocating fixed `-Xms12g -Xmx16g` memory) and sets JMH parameters exactly to `10` forks and `3000` iterations in SingleShotTime mode.
* **`run_finals_regression.sh`**: Exactly identical in structure and parameters to the `no_regression` script, but explicitly targets the code state where the regression was injected. Both execution scripts automatically generate and store `.json` output files in the `results/` folder.

### Data Analysis and Visualization
Because JMH outputs require rigorous statistical processing to differentiate noise from actual regressions, these Python scripts process the `.json` output files directly.
* **`hierarchical_bootstrap.py`**: The main statistical engine. It extracts the raw execution data from the JSON files and performs a Hierarchical Bootstrap resampling. This correctly accounts for the correlation between iterations within the same JVM fork, generating the final Confidence Intervals (CIs) and performance p-values.
* **`visualize_results.py`**: Transforms the complex statistical outcomes generated by the bootstrap script into readable distribution charts, box plots, and graphs using `seaborn` and `matplotlib`. The output is dumped into the `visualizations/` folder.

## Execution Order
To correctly apply this methodology:
1. Run `optimize_system.sh`.
2. Ensure the code is clean (using `prepare_no_regression.sh` if needed).
3. Run `run_finals_no_regression.sh` to obtain baseline data.
4. Run `inject_regression_code.sh` to introduce artificial stress.
5. Compile changes with Maven.
6. Run `run_finals_regression.sh` to obtain the regression data.
7. Run `restore_system.sh`.
8. Execute `hierarchical_bootstrap.py` followed by `visualize_results.py` on the resulting datasets.
