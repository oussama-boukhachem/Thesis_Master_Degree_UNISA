# Benchmark Sources

This directory contains the Java source code for all JMH (Java Microbenchmark Harness) microbenchmarks evaluated in the thesis. To comply with file size limitations while ensuring complete reproducibility, we provide only the benchmark files rather than the entire Apache Ignite repository.

## Directory Structure

The sources are strictly compartmentalized by the specific Apache Ignite commit hashes used in the experimental setup:

* **commit1_b038730** (corresponds to Apache Ignite v1.8.0)
* **commit2_feba953** (corresponds to Apache Ignite v2.0.0)
* **commit3_160dab0** (corresponds to Apache Ignite v2.2.0)

Inside each commit directory, the files strictly follow the standard Apache Ignite module structure:
`modules/benchmarks/src/main/java/org/apache/ignite/internal/benchmarks/jmh/`

## Benchmark Categories

Within the `jmh` package, the files are divided into two primary categories corresponding to the experimental groups in the study:

### 1. Handwritten Benchmarks
These are the original developer-written benchmarks that serve as the control group. They are located in their respective semantic packages:
* `cache/`: Cache operations and locking mechanisms (e.g., `JmhCacheBenchmark.java`, `JmhCacheLocksBenchmark.java`).
* `future/`: Asynchronous and future operations (e.g., `JmhFutureAdapterBenchmark.java`).
* `misc/`: Atomic operations and other utilities (e.g., `JmhIncrementBenchmark.java`).
* `notify/`: Thread parking and wait strategies.

### 2. AI-Generated Benchmarks
These are the experimental benchmarks generated by the Large Language Model, developed to be functionally equivalent twins of the handwritten versions. 
* All AI-generated benchmarks are isolated within the `ai/` package directory (e.g., `JmhCacheBenchmarkAI.java`, `JmhIncrementBenchmarkAI.java`).

## Usage Instructions

To run these benchmarks within the actual Apache Ignite environment:

1. Clone the Apache Ignite repository.
2. Checkout the specific commit hash (like `git checkout b038730`).
3. Copy the `modules/` directory from the corresponding commit folder in this repository and paste it into the root of your checked-out Apache Ignite repository. Therefore, this will place the `.java` files in the exact locations expected by the Maven build system.
4. Continue with compilation as defined in the main repository instructions.
